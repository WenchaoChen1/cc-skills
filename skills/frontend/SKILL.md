# /dev/frontend — 前端开发

> 基于设计文档实现前端代码。
> 公共规则见 `/dev/common`，本文件仅定义前端特有的实现流程。
> 工作目录：由 `CLAUDE.md` 中的前端项目路径决定。

## 使用方式

```
/dev/frontend <功能名称>
```

> **独立调用时**：自动执行 `/dev/common` 的准备阶段（读 CLAUDE.md → 读设计文档 → 读需求文档 → 探索前端代码模式）。
> **由 `/dev/run` 调度时**：准备阶段已由调度器完成，直接进入实现步骤。

---

## 前置：从 CLAUDE.md 和现有代码提取项目信息

开始实现前，必须确认以下信息（**不得使用预设值**）：

| 需确认项 | 获取方式 |
|---------|---------|
| 前端框架 | `CLAUDE.md` 或 `package.json`（React/Vue/Angular/Next.js 等） |
| UI 组件库 | `package.json`（Ant Design/Element/MUI/Vuetify 等） |
| 状态管理 | 现有代码（Redux/MobX/dva/Pinia/Vuex/useState 等） |
| 请求工具 | 现有 service 文件（axios/fetch/umi-request 等） |
| 语言 | `tsconfig.json` 或现有代码（TypeScript/JavaScript） |
| 路由配置位置 | `CLAUDE.md` 或搜索项目中的路由配置文件 |
| 样式方案 | 现有代码（CSS Modules/styled-components/Tailwind/Less/Sass 等） |
| 页面目录约定 | `CLAUDE.md` 或现有页面文件所在目录 |
| API service 目录 | 现有 service 文件所在目录 |
| 类型定义位置 | 现有 interface/type 文件所在目录 |

## 实现步骤

按设计文档第 2、3 章（UI + 前端实现要点）逐一实现。

### 2-A. API Service 层

- 位置：参照现有 service 文件的目录和命名
- 为设计文档每个接口写一个请求函数
- 使用项目现有的请求工具和封装方式
- 参照现有 service 文件的导出方式和错误处理

**类型定义**（若项目使用 TypeScript）：
- 位置：参照现有类型文件的存放位置（与 service 同文件 / 独立 `typings` 文件 / 其他）
- 请求参数类型 + 响应数据类型
- **字段名必须与后端 DTO/Schema 完全一致**

### 2-B. 页面组件

按设计文档第 3.1 节的文件结构创建目录和文件。

**实现时逐条对照设计文档第 2 章**：

| 设计文档章节 | 实现内容 |
|-------------|---------|
| **2.1 页面布局** | DOM/组件结构，间距比例 |
| **2.2 组件清单** | 逐一实现每个组件 |
| **2.3 数据展示规则** | 数字格式化、颜色语义、空值显示 |
| **2.4 交互行为** | 每条交互逐一实现，**不遗漏** |
| **2.5 空态/加载态** | Loading、空数据提示、错误态，三种状态均处理 |

**页面入口组件**：
- 负责数据加载和顶层状态管理（使用项目现有的状态管理方式）
- 筛选条件变更时刷新数据
- 处理 loading / error / empty 三种状态

**子组件**：
- 按设计文档组件清单逐个实现
- Props 类型明确定义（若 TypeScript）
- 不在子组件中直接调用 API（由父组件传入数据和回调）

**样式**：
- 使用项目现有的样式方案
- 颜色/间距参照设计文档第 2.3 节约定
- 不引入额外的样式框架

### 2-C. 路由注册

- 找到项目的路由配置文件，在合适位置插入新路由
- 权限/鉴权字段参照同类已有路由，不自行添加未见过的属性
- 路由路径与设计文档第 3.1 节一致

### 2-D. 自检

完成后逐项确认：
- [ ] 类型定义无错误（interface/type 字段名与后端一一对应）
- [ ] 所有 API 调用路径与后端接口路径一致
- [ ] 设计文档第 2.4 节的**每条交互行为**都有对应实现
- [ ] 设计文档第 7 章的每个异常场景都有前端处理
- [ ] 空态和加载态已实现（Loading + Empty + Error）
- [ ] 路由已注册，页面可访问
- [ ] 数字格式化规则与设计文档一致
- [ ] 无未使用的 import

## 前端常见错误预防

| 错误类型 | 预防措施 |
|---------|---------|
| 类型字段名与后端不一致 | 逐字段核对后端 DTO/Schema 或数据存档 |
| 表格 colSpan/rowSpan 计算错误 | 行计数函数覆盖所有行类型 |
| 路由权限配置报错 | 参照同类已有路由，不自行添加未见过的属性 |
| 数字格式化不一致 | 先搜索项目是否已有格式化工具函数 |
| 组件 key 缺失 | 列表渲染使用唯一且稳定的 key（优先 id） |
