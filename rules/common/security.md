# 安全准则

## 必须执行的安全检查

每次提交前：
- [ ] 无硬编码密钥（API keys、密码、tokens）
- [ ] 所有用户输入已校验
- [ ] SQL 注入防护（参数化查询）
- [ ] XSS 防护（HTML 已转义）
- [ ] CSRF 防护已启用
- [ ] 认证/授权已验证
- [ ] 所有端点已设置速率限制
- [ ] 错误消息不泄露敏感数据

## 密钥管理

- 绝不在源代码中硬编码密钥
- 始终使用环境变量或密钥管理器
- 在启动时验证所需密钥是否存在
- 轮换任何可能已泄露的密钥

## 安全响应协议

发现安全问题时：
1. 立即停止
2. 使用 **security-reviewer** agent
3. 在继续之前修复 CRITICAL 级别问题
4. 轮换任何已暴露的密钥
5. 审查整个代码库中的类似问题
